WISSENSBASIS (aus „Kapitel 1 – Einführung“, vereinfacht)
────────────────────────────────────────────────────────
- Docker ist ein Tool. Es hilft, Programme als **Paket** zu „verpacken“.
  Dadurch kann man Software einfacher **ausliefern (Deployment)** und in vielen Umgebungen ähnlich laufen lassen.
- Container sind **keine** klassischen virtuellen Maschinen (VMs).
  - VM: „Computer im Computer“. Hat ein eigenes Betriebssystem. Läuft über einen Hypervisor.
  - Container: Teilt sich den Kernel mit dem Host. Ist oft leichter. Braucht oft weniger Ressourcen.
- Docker macht Container einfacher nutzbar durch:
  - eine einfache Kommandozeile (CLI)
  - ein Image-Format / Standard, damit Builds reproduzierbar sind.
- Wichtige Begriffe:
  - Docker-Client: der Befehl `docker` (das, was man tippt)
  - Docker-Server: `dockerd` (läuft im Hintergrund und macht die Arbeit)
  - Docker-Image: „Vorlage/Paket“. Besteht aus Layers + Metadaten. Hat Name + Tag (Tag = Version).
  - Container: ein gestartetes Image (laufendes Programm)

ZIELBILD
Die Lernenden können Container und „klassische“ Virtualisierung (VM) grob unterscheiden
und den Nutzen in der Praxis erklären (Auswirkungen der Virtualisierung erkennen).

RAHMEN
- Dauer: 90 Minuten
- Niveau: Anfänger
- Methode: kurzer Input + viel Praxis + Partnerarbeit
- Material: PC mit Docker installiert, Beamer/Whiteboard, Internet (optional)
- Wichtig: Beobachten und erklären können. Nicht auswendig lernen.

────────────────────────────────────────────────────────
KAPITEL 1 (0–10 Minuten): Einstieg mit Alltagsanalogie + Erwartungen
────────────────────────────────────────────────────────
- Starte mit einer kurzen Geschichte (sehr einfach):
  „Eine App läuft bei mir, aber bei dir nicht. Warum? Weil dein Computer anders ist.“
- Frage die Klasse:
  1) „Wer hatte das schon mal?“
  2) „Was könnte der Grund sein?“ (z. B. andere Version, andere Einstellungen, fehlt etwas)
- Sage dann:
  „Heute lernen wir eine Lösung kennen: Container. Das ist wie ein Paket, das fast überall gleich läuft.“
- Sammle 2–3 Erwartungen: „Was wollt ihr heute können?“
- Schreibe die Erwartungen kurz an die Tafel.

────────────────────────────────────────────────────────
KAPITEL 2 (10–25 Minuten): Input (kurz): Docker‑Versprechen + Begriffe (einfach)
────────────────────────────────────────────────────────
- Erkläre in einfachen Worten 3 Dinge, die Docker verspricht:
  1) „Ein Programm kommt als Paket. Es läuft dann in vielen Umgebungen gleich.“
  2) „Es startet oft schnell, weil es weniger ‚Ballast‘ hat als eine ganze VM.“
  3) „Es ist getrennt vom restlichen PC. Das hilft Ordnung zu halten.“

- Erkläre diese Begriffe (mit je 1 Mini‑Beispiel):
  - Docker (Werkzeug): „Damit starten wir Container.“
  - Image (Paket/Vorlage): „Das ist das Paket. Daraus startet man Container.“
  - Container (laufendes Programm): „Das ist das gestartete Paket.“
  - Registry (Online‑Lager): „Dort liegen Images. Wenn das Image fehlt, kann Docker es holen.“
  - Tag (Version): „Ein Tag ist wie eine Versionsnummer vom Image.“

- Baue kurz ein (sehr einfach, 1–2 Sätze):
  - Docker‑Client: „Was ich tippe: docker …“
  - Docker‑Server: „Im Hintergrund: dockerd …“

- Erkläre den Unterschied VM vs Container in sehr einfachen Worten (2–3 Sätze):
  - VM: „Ein ganzer Computer im Computer. Mit eigenem Betriebssystem. Ein Hypervisor verwaltet das.“
  - Container: „Teilt sich den Kernel mit dem PC. Darum ist er oft leichter und schneller.“
  - Merksatz: „VM = ganzes OS. Container = Prozess mit Paket, teilt Kernel.“

────────────────────────────────────────────────────────
KAPITEL 3 (25–65 Minuten): Geführte Mini‑Demo im Terminal („hello-world“)
────────────────────────────────────────────────────────
Gib ein genaues **Lehrerskript** mit:
- den Befehlen
- den erwarteten Beobachtungen
- kurzen Fragen an die Lernenden

Schritte:
1) Lehrperson tippt im Terminal:
   docker run hello-world

2) Lernende beobachten: Was passiert?
   - Erwartung: Es wird Text ausgegeben.
   - Es kann sein, dass Docker zuerst etwas „lädt“.

3) Erkläre in einfachen Worten die 3 Schritte:
   - „Docker sucht das Image.“
   - „Wenn es fehlt, lädt Docker es aus einer Registry (Online-Lager).“
   - „Docker startet einen Container und führt das Programm aus.“

4) Zeige danach:
   docker images
   docker ps -a

5) Frage die Lernenden (kurz, einfach):
   - „Seht ihr das Image in der Liste?“
   - „Seht ihr den Container in der Liste?“
   - „Läuft der Container noch oder ist er fertig?“
   - „Warum ist er fertig?“ (Weil das Programm nur kurz etwas ausgibt und endet.)

6) Optional (wenn Zeit, nur wenn sinnvoll):
   docker rm <CONTAINER_ID>
   Erkläre: „Wir löschen nur den Container-Eintrag. Das Image bleibt.“

WICHTIG: Während du das Tutorial ausführst:
- Nach JEDEM Befehl: Pause + Frage an den User, ob es geklappt hat.
- Wenn nein: gib einfache Troubleshooting-Schritte.

────────────────────────────────────────────────────────
KAPITEL 4 (65–80 Minuten): Partnerauftrag – Mini‑Glossar (5 Begriffe)
────────────────────────────────────────────────────────
Auftrag:
- Bildet Zweiergruppen.
- Schreibt 5 Begriffe auf.
- Zu jedem Begriff: 1 Satz Erklärung + 1 Beispiel (1 Satz).
- Nutzt einfache Worte. Kein Fachchinesisch.

Gib die 5 Begriffe und fertige Beispielsätze **genau so** aus:

1) Docker
   - Erklärung: „Docker ist ein Werkzeug, das Programme als Container starten kann.“
   - Beispiel: „Mit ‚docker run hello-world‘ starte ich einen Container.“

2) Image
   - Erklärung: „Ein Image ist ein Paket als Vorlage. Es enthält Dateien und Infos, die ein Programm braucht.“
   - Beispiel: „Das Image ‚hello-world‘ kann ich auf vielen PCs nutzen.“

3) Container
   - Erklärung: „Ein Container ist ein gestartetes Image. Er ist das laufende Programm.“
   - Beispiel: „Der hello-world-Container startet, zeigt Text und stoppt wieder.“

4) Registry
   - Erklärung: „Eine Registry ist ein Online-Lager, wo Images gespeichert sind.“
   - Beispiel: „Wenn ein Image fehlt, kann Docker es aus einer Registry herunterladen.“

5) Virtuelle Maschine (VM)
   - Erklärung: „Eine VM ist ein kompletter Computer im Computer, mit eigenem Betriebssystem.“
   - Beispiel: „Eine VM braucht oft mehr Speicher als ein Container.“

────────────────────────────────────────────────────────
KAPITEL 5 (80–90 Minuten): Exit‑Ticket (3 Fragen) + Musterantworten
────────────────────────────────────────────────────────
Gib 3 kurze Fragen, die das Zielbild prüfen. Gib die Antworten direkt darunter.

Frage 1: „Was ist der Unterschied zwischen Image und Container?“
Antwort: „Ein Image ist die Vorlage (Paket). Ein Container ist das gestartete, laufende Programm aus diesem Paket.“

Frage 2: „Warum braucht eine VM oft mehr Ressourcen als ein Container?“
Antwort: „Weil eine VM ein eigenes Betriebssystem hat und über einen Hypervisor läuft. Ein Container teilt den Kernel und ist oft leichter.“

Frage 3: „Was ist ein Tag bei einem Image?“
Antwort: „Ein Tag ist wie eine Versionsnummer. Er hilft, eine bestimmte Version vom Image zu wählen.“
